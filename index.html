python_.venv虚拟环境的使用

发表于2021-05-17更新于2021-05-18字数统计635

python虚拟环境 *.venv*:
在Python3.3以上的版本中，python标准库中自带了venv工具。在之前的版本中使用虚拟环境需要安装virtualenv包。
venv工具就是一个用来管理python运行环境的本地虚拟环境（local virtual environment）。它创建了一个独立的易于复制的python运行环境（特定的版本的包、依赖等。detach from the operating system interpreter and environment），避免了因为包的版本的区别、包之间的冲突等运行环境的差异带来的错误。

venv 环境的激活

python3.3以上版本虚拟环境venv激活

  1. 创建并进入项目文件夹

  2. 创建包含虚拟环境的子文件夹

    可以看到my-dir文件夹下新增了.venv文件夹，该文件夹中包含了python的虚拟环境

  3. 激活虚拟环境

  • 未激活之前我们可以看一下当前系统中的python路径
  • 执行.venv\Script文件夹中的activate脚本来激活虚拟环境
  • 此使虚拟环境就已经激活了。再查看当前系统python路径，可以看到python已经切换为虚拟环境中的python编译器了

pip requirements.txt进行外部包的安装

python的包资源十分丰富。python的官方包目录（https:\pypi.org）。
通过requirements文件对包、包版本等进行管理，可以实现python依赖环境的可复制（replicable），避免因为依赖版本不一致带来的编译错误。

  • 新建requirements.txt文件 在项目文件夹新建一个requirements.txt文件，文件中包含需要安装的包的名称和版本，未指定版本的安装时会安装最新版本。
  • pip install
    激活虚拟环境，使用python自带的pip工具安装第三方包。-r 指定文件名。 -i 指定下载源为国内源，默认的国外源下载速度非常慢。 $ pip list可以查看已安装的包列表，显示出的列表可能比我们填写在requirements文件中的包数量要多，这是因为pip安装了这些包需要的依赖包。
    $ pip freeze > requirements.txt将已安装的包信息（包含依赖）写入到requirements.txt文件中。
