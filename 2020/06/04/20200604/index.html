hexo框架博客部署到github repository

Created2020-06-04Updated2020-06-05

通过hexo创建博客后，将博客文章生成的静态网页文件部署到github仓库上，通过github仓库的github page就可以访问这个博客站点。

Hexo 创建一篇博客文章

在hexo根目录下，打开控制台。输入 hexo new filename运行。 在soure\_posts文件夹下就创建了一个filename.md的markdown文件。

Hexo 生成静态文件

编辑完博客文章后，可以使用hexo s或hexo server命令启动一个本地server。在浏览器中可以打开这个本地地址http://localhost:4000 来预览博客的效果。

启动本地hexo服务

生成浏览器能解析的静态html文件。 控制台运行hexo g或hexo generate命令。在hexo根目录下会生成一个public文件夹，里面的文件就是hexo自动生成的静态网页文件。

生成静态文件

Github 部署Hexo博客

  1. github上新建一个仓库用于存放博客网页的静态文件。

    新建一个repository

    repository命名 
    注意：仓库必须命名为"账户名.github.io"的形式。新建好了仓库之后就可以部署hexo文件了。

  2. hexo文件部署

    进入hexo根目录，安装hexo-deployer-git。

    安装完成后，打开hexo根目录下的_config.yml文件，编辑最下方的deploy配置。

    repository:后面就是github中创建的仓库的gitpage链接。
    编辑完成并保存_config.yml文件后就可以把静态文件部署到github仓库了。
    hexo g生成静态文件。 hexo d或git deploy命令部署博客文件。
    部署完成后，在上面创建的github仓库中就可以看到与本地hexo根目录下的public文件夹内相同的文件。

  3. 访问博客
    在浏览器中输入gitpage的链接就可以访问博客主页了。
