hexo框架博客部署

发表于2020-06-04更新于2020-06-13字数统计1.4k

通过hexo创建博客后，将博客文章生成的静态网页文件部署到github仓库上，通过github仓库的github page就可以访问这个博客站点。

Hexo 创建一篇博客文章

在hexo根目录下，打开控制台。输入 hexo new filename运行。 在soure\_posts文件夹下就创建了一个filename.md的markdown文件。

Hexo 生成静态文件

编辑完博客文章后，可以使用hexo s或hexo server命令启动一个本地server。在浏览器中可以打开这个本地地址http://localhost:4000 来预览博客的效果。

启动本地hexo服务

生成浏览器能解析的静态html文件。 控制台运行hexo g或hexo generate命令。在hexo根目录下会生成一个public文件夹，里面的文件就是hexo自动生成的静态网页文件。

Github Pages 自定义博客网址

上面方法实现的部署在Github仓库的博客的网址是用户名.github.io的形式。如果想要设置为自己想要的网址，可以通过Github pages中的custom domain（自定义域名）实现。

在custome domain中输入自己的域名并保存

  1. 注册域名
    要给博客自定义域名，首先需要有一个自己的域名。购买域名的平台有很多，我使用的是阿里云的域名服务。

    注册阿里云账号后，进入控制台-域名注册页面搜索自己想要的域名。click 。注册购买完成后需要实名认证才能使域名生效。

  2. 关联Github pages与自定义域名
    完成域名注册后，接下来就是要把存放博客静态文件的github仓库的github page与自定义的域名关联起来了。
    第一步 首先确定博客github pages的IPV4地址。 打开控制台，ping博客的github网页地址。

    记下IP地址 185.199.108.153
    第二步 域名解析。 阿里云控制台->域名列表->域名操作->解析

    添加一个A类型记录。 主机记录可以为默认值@或www。 记录值填写上面ping的地址，185.199.108.153

    添加完成后勾选启用，域名解析配置完成。

    第三步 github仓库设置github pages的custome domain(自定义域名)。 把自定义域名设置为自己注册的域名,点击保存。

    设置并保存之后，就可以通过自定义域名访问博客了。
    第四步 完成以上步骤之后，在github博客仓库内会生成一个CNAME文件。文件的内容是上面第三步填写的自定义域名地址。但是本地文件中没有这个CNAME文件，如果下次再次部署博客，这个文件会被覆盖掉，导致无法通过自定义链接访问博客。因此需要在本地文件夹hexo根目录下的source文件夹内新建一个CNAME文件，文件内容是自定义的域名地址。

Github + jsDelivr免费图床搭建和PicGO的使用

我们在写博客的时候，经常需要添加一些图片，图床就是指存储图片的服务器。
jsDelivr是一个免费、开源的加速CDN公共服务。托管了许多大大小小的项目，可加速访问托管的项目目录或图片资源。将图片文件部署在github仓库，通过jsdelivr加速访问是一种很方便的免费图床方案。

  1. github创建图床仓库并生成TOKEN

    进入设置页面, 创建一个ACCESS TOKEN





    上面生成的一串代码就是access token, 先把它复制保存下来，后面配置PicGo软件的时候会用到。
  2. PicGo软件安装与配置
    PicGo是一个用于快速上传并获取图片url的工具，支持七牛图床、腾讯云、阿里云、GitHub、Imgur等多个图床。在PC上截图后，使用PicGo的快捷键CTRL+SHIFT+P可以将图片上传到这些图床上，并且还会自动复制一个markdown格式的图片地址。这样就可以直接黏贴在markdown文本中快速地访问图片。
